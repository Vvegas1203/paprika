  1. ОБЩАЯ АРХИТЕКТУРА
================================================================================

Проект представляет собой монолитное SPA-приложение для изучения немецкого
языка. Основной принцип: изучение слов через карточки (свайп-механизм) +
теоретический блок по грамматике с практическими заданиями.

1.1. Технологический стек
─────────────────────────
  - React 18 с функциональными компонентами и хуками
  - TypeScript (строгая типизация, discriminated unions, brand types)
  - Vite 6 (сборка, HMR, оптимизация)
  - React Context (управление языком интерфейса)
  - localStorage (персистентность прогресса)
  - CSS Custom Properties (темизация)
  - CSV-парсер (загрузка словарных данных)

───────────────────────────────

  - После ответа: показать правильный ответ при ошибке

2.7. BlitzResult.tsx / TopicResult.tsx
────────────────────────────────────────
  - Круг SVG с процентом
  - Статистика: верно, неверно, всего

2.8. GrammarLevelList.tsx (Выбор уровня грамматики)

2.9. GrammarTopicList.tsx (Выбор темы грамматики)
─────────────────────────────────────────────────

  - Прогресс по темам (завершённые уроки %)
  - Оптимизация через useMemo

2.10. GrammarLesson.tsx (Урок грамматики — теория)
───────────────────────────────────────────────────
  - Отображает: вступление, правила (с формулами), примеры,
    частые ошибки, шпаргалку, итоги
  - Badge завершения (зелёная галочка)
  - Кнопка "Практика" / "Завершено"
  - getText() для локализации любого поля

2.11. GrammarPractice.tsx (Практика — задания)
───────────────────────────────────────────────
  - Два режима: 'tasks' (новый TaskCollection) и 'exercises' (старый)
  - Авто-определение типа задания (getItemKind)
  - Поддержка 7 типов:
    • choice — множественный выбор (кнопки-опции)
    • scenario — выбор реакции на ситуацию
    • fill — заполнение пропуска (input)
    • word-order — сборка предложения (input + подсказка)
    • parts-order — сборка частей (input + подсказка)
    • comparison — таблица степеней сравнения
    • transform — трансформация (input)
  - Обратная связь: верно/неверно + правильный ответ при ошибке
  - Навигация: Next → Finish

  3. ГРАММАТИЧЕСКИЙ МОДУЛЬ (CORE)
================================================================================

3.1. Архитектура
─────────────────
  GrammarEngine — центральный класс, реализующий паттерн Facade.
  Использует провайдеры (IContentProvider) для получения контента,
  ProgressTracker для отслеживания прогресса, AnswerChecker для
  проверки ответов, TypedEventEmitter для событий.

  Событийная модель:
    engine:initialized    — движок готов
    exercise:answered     — пользователь ответил
    progress:updated      — прогресс изменился
    topic:completed       — тема завершена
    error                 — ошибка провайдера

3.2. Типы данных
─────────────────
  Базовые:
    CefrLevel: 'A1' | 'A2' | 'B1' | 'B2' | 'C1' | 'C2'
    GermanCase: 'nominativ' | 'akkusativ' | 'dativ' | 'genitiv'
    Gender: 'maskulin' | 'feminin' | 'neutrum' | 'plural'
    Person: '1sg' | '2sg' | '3sg-m' | ... | 'formal'
    Tense: 'prasens' | 'perfekt' | 'prateritum' | 'futur1'
    Slug: brand type (string & { __brand?: 'Slug' })
    Localized<T>: Partial<Record<LangCode, T>> & { ru: T }
    Difficulty: 1 | 2 | 3 | 4 | 5

  Тема (GrammarTopic):
    id, level, category, title, description, order,
    prerequisites?, introduction, rules[], examples[],
    mistakes[], tips[], cheatsheet?, meta?

  Упражнения (Exercise — discriminated union):
    8 видов: choice, fill, match, transform, translate,
    conjugate, decline, spot-error, order-words
    Каждый с уникальными полями (options, template, answer...)

  Практические задания (TaskCollection):
    id, level, category, order, title, description?, tasks[]
    Task: { id, title, instruction, items[] }
    7 видов items: choice, fill, word-order, parts-order,
    comparison, transform, scenario

3.3. Провайдеры
────────────────
  IContentProvider — интерфейс для sources контента:
    loadTopics(), loadExercises(), loadTaskCollections()

  BuiltInProvider — встроенный статический провайдер.
  Архитектура позволяет добавлять:
    - RemoteProvider (API)
    - JsonFileProvider (JSON-файлы)
    - DatabaseProvider (SQLite/Postgres)

3.4. Прогресс
─────────────
  ProgressTracker:
    - TopicProgress: completion %, score, exercises done
    - OverallProgress: общая статистика по уровню
    - ProgressStorage: load(), save(), clear()
    - InMemoryStorage — реализация по умолчанию

  useGrammarProgress (хук для UI):
    - lessonProgress (Record<string, UserGrammarProgress>)
    - topicProgress (Record<string, TopicGrammarProgress>)
    - recordLessonComplete, recordExerciseComplete
    - getOverallGrammarProgress

  useProgress (хук для карточек):
    - progress (Record<number, 'known' | 'learning'>)
    - streak, weekStats, dayStats
    - getKnownCount, isFullyKnown
    - getTopicTestProgress, markTopicTestWordCorrect

  7. СЛАБЫЕ СТОРОНЫ И РИСКИ (МИНУСЫ)
================================================================================

  7.1. Критические:
  ──────────────────
  ✗ Нет роутинга: вся навигация через useState + if-else в App.tsx
    — с ростом числа экранов станет неуправляемым
  ✗ Нет серверной части: всё в localStorage — потеря данных при
    очистке кэша, нет синхронизации между устройствами
  ✗ Нет тестов: ни unit, ни integration, ни e2e
  ✗ Нет бэкенда: нельзя сохранять прогресс пользователя

  7.2. Архитектурные:
  ────────────────────
  ✗ App.tsx — God-компонент: 589 строк, 50+ useState, вся логика
  ✗ Все стили в одном файле (App.css, 2000+ строк)
  ✗ Нет CSS Modules / CSS-in-JS — риск коллизий имён
  ✗ Нет code splitting — весь бандл 284KB (нормально для старта)
  ✗ Нет сервис-воркера — не работает офлайн

  7.3. Функциональные:
  ─────────────────────
  ✗ Нет регистрации/авторизации
  ✗ Нет платёжной системы
  ✗ Нет профиля пользователя
  ✗ Нет системы уровней/геймификации (кроме streak)
  ✗ Нет рекомендательной системы (какую тему учить следующей)
  ✗ Нет уведомлений (push/email)
  ✗ Нет социальных функций (друзья, рейтинги)
  ✗ Нет адаптивности под планшеты/десктоп (только mobile 480px)
  ✗ Нет доступности (a11y): aria-labels, keyboard nav

  7.4. Контент:
  ─────────────
  ✗ Только уровень A1 (нет A2, B1...)
  ✗ Нет аудио (озвучка слов, произношение)
  ✗ Нет изображений в уроках (кроме карточек слов)
  ✗ Нет видео/интерактива
  ✗ Практические задания без полной проверки семантики
    (сравнение строк, не учитываются альтернативы)

  8. ФУНДАМЕНТ ДЛЯ ДАЛЬНЕЙШЕГО РАЗВИТИЯ

  8.1. Хорошо заложено:
  ──────────────────────
  ✓ Модульная архитектура: грамматика как отдельный модуль
    позволяет добавлять новые уровни (A2, B1...) простым созданием

  ✓ Provider pattern: для подключения бэкенда достаточно реализовать
    IContentProvider с HTTP-клиентом
  ✓ ProgressStorage: можно заменить InMemoryStorage на
    LocalStorageAdapter / RemoteStorageAdapter
  ✓ Типизированное ядро: добавление новых типов упражнений —
    просто добавить интерфейс + вид в discriminated union
  ✓ i18n: добавление языка — создать файл в i18n/ + добавить
    переводы
  ✓ Builders: TopicBuilder упрощает добавление контента —
    не нужно знать структуру GrammarTopic

  8.2. Требует переработки:
  ──────────────────────────
  ✗ Роутинг: необходим React Router v7 (или TanStack Router)
  ✗ State management: App.tsx нужно разбить на страницы с
    изолированным состоянием (Zustand / Redux Toolkit)
  ✗ Стили: перейти на CSS Modules или Tailwind для изоляции
  ✗ Адаптивность: переделать все размеры с px на rem/em,
    добавить media queries для планшетов и десктопа
  ✗ Code splitting: React.lazy + Suspense для каждого экрана
  ✗ Тесты: Vitest + React Testing Library
